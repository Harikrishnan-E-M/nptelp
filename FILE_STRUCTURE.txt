🚀 QUICK START COMMANDS

Manual Setup (All platforms):
  cd backend && npm install
  cd ../frontend && npm install

Start Services (3 terminals):

Open in Browser:

  Sanity Studio: http://localhost:3333

BACKEND (Express.js on Port 5000)
├─ Handles all API requests
├─ Manages CSV file uploads
├─ Parses CSV data

├─ Calculates statistics
└─ Returns JSON responses

FRONTEND (React + Vite on Port 3000)
├─ Shows academic years as cards

├─ Handles CSV file upload UI

├─ Calls backend API
└─ Uses Bootstrap for styling

SANITY STUDIO (Port 3333)
├─ Web interface to Sanity CMS
├─ Manage academic years directly
├─ Manage NPTEL data records
├─ View all documents
├─ Edit schema
└─ Query data with GROQ

🌐 API ENDPOINTS REFERENCE

GET /api/years
  Returns: Array of all academic years
  Example: [{ _id: "...", yearLabel: "2023-24", dataCount: 150 }]

POST /api/years
  Body: { yearLabel, startYear, endYear, description }
  Returns: Created year document

DELETE /api/years/:yearId
  Returns: { success: true }
  Note: Deletes all data for that year

GET /api/statistics/:yearId
  Returns: {
    totalStudents: number,
    passCount: number,
    failCount: number,
    pendingCount: number,
    averageScore: string,
    batches: string[],
    semesters: string[],
    data: nptelData[]
  }

POST /api/upload/:yearId
  Body: FormData with file
  Returns: { success: true, recordCount: number }

GET /health
  Returns: { status: "ok" }

🆘 QUICK TROUBLESHOOTING

Issue: Port 5000 already in use
Solution: Change PORT in backend/.env

Issue: Frontend can't connect to backend
Solution: Ensure backend is running on 5000

Issue: Sanity token invalid
Solution: Get new token from manage.sanity.io and update .env

Issue: CSV upload fails
Solution: Check CSV column names match requirements

Issue: "Cannot find module"
Solution: Run npm install in that directory
